# EEG 储层计算预警项目 (EEG-RC Pre-ictal Warning)

这是一个基于 CHB-MIT 头皮脑电图 (EEG) 数据库的异常预警流水线。本项目将**基于忆阻器的储层计算网络 (Memristor Reservoir Computing)** 适配到多通道时间序列 EEG 数据上，并结合深度学习解码器，构建一个面向临床预警场景的工程化 `pre-ictal` 检测系统。

> 说明：
> 本项目当前的 `pre-ictal` 是工程定义，而不是严格的“预测未来 15s 发作”的纯时间学定义。
> 在当前器件和编码能力约束下，我们采用了一条更务实的路线：保留原始 RC 主体与原版脉冲风格，让系统优先学会“尽可能早地把异常脑电从正常背景里拉开”，从而把原先的 seizure-oriented pipeline 调整为具有提前预警意义的异常告警系统。

---

## 🔄 从 Seizure 到 Pre-ictal 的重定义

本项目最初的任务是**癫痫发作检测**，但在与导师讨论后，我们将目标调整为**预警**。

这里的转变不是简单把图里的 `Seizure` 改名成 `Pre-ictal`，而是以下工程层面的重新定位：

1. **目标从“是否已经发作”转为“是否进入危险异常状态”**  
   我们不再追求严格的长时未来预测，而是希望系统在发作真正完全建立之前，就已经对异常脑电给出明显响应。

2. **不推翻原有 RC 主体，而只微调编码规则**  
   忆阻器储层最擅长利用“脉冲密度差”和“自然不规则性”进行区分，因此我们没有改动 RC 主体、特征维度和 1D CNN 主架构，而是只在脉冲编码上做最小改动。

3. **保留原版最有效的脉冲风格**  
   实验表明，原版系统之所以好分，不是因为复杂的人为事件规则，而是因为它天然形成了：
   - `Normal`：稀疏、留白多、局部静默
   - `Positive / Pre-ictal-like`：更密、更连续、热图更亮  
   因此我们最终放弃了规则化打点，回到原版 4 通道脉冲结构，只增加了对局部异常的门控。

4. **当前版本强调“工程可用的提前告警”**  
   从项目定位上，我们将当前正类解释为 `pre-ictal-like warning state`。这一定义更贴合器件实际能力，也更符合本项目“先把预警做出来”的目标。

## 🌟 核心特性与解决的痛点

1. **可变通道适配机制 (频带聚合)**: 
   不同患者（甚至同一患者的不同记录）的通道数不固定（如 23 或 24 个）。本方案首先对每个通道进行**频带分解** (Delta, Theta, Alpha, Beta)，然后通过**Max-Abs Pooling (最大绝对值池化)** 进行跨通道聚合。这不仅将任意通道数固定为了 `(4, time_steps)`，还保留了局部脑区可能存在的瞬间癫痫高频爆发。

2. **面向预警的最小编码改造 (Fixed Absolute Threshold + Relative Gating)**:
   为了从原本的 seizure-oriented 方案平滑过渡到 pre-ictal warning，本项目没有推翻原版编码框架，而是在原有**固定绝对阈值 + 不应期**的基础上，增加了**局部相对异常门控**。只有当当前偏离既超过固定阈值，又显著高于局部背景波动时才触发 spike。  
   这一改动保留了原版“正常稀疏、异常密集”的自然脉冲形态，同时让系统对 pre-ictal 阶段较弱但持续的异常波动更敏感。

3. **严格保留时序的 RC 特征提取**:
   忆阻器储层网络作为一个“无监督的漏积分器物理系统”，按时间步处理脉冲。为防止电流截断饱和，本项目采用了**指数级状态跃迁与基线相对衰减**，确保储层状态能够真实反映脉冲序列的短时记忆。

4. **1D CNN 深度特征解码器**:
   鉴于原始 RC 提取的物理特征在应对复杂生理噪声时仍处于高度纠缠状态（t-SNE难以区分），本项目放弃了简单的线性分类器，引入了 **1D CNN (一维卷积神经网络)**。它将 1024 维特征重塑回时空维度，通过滑动卷积成功解开了高维流形，并在隐层空间形成了清晰的癫痫/正常聚类簇。

5. **时序平滑后处理 (Sliding Window Smoothing)**:
   考虑到脑电切片（2秒/窗口）存在孤立的假阳性判断，本项目在网络预测后引入了**滑动窗口多数投票机制 (Window Size = 5)**。这一策略结合了异常状态持续性的先验，成功过滤了大部分孤立误报，使预警结果更稳定。

6. **流式数据加载与分层验证 (Anti-OOM & Data Leakage)**:
   - **内存管理**：采用“逐患者落盘处理”和 `PatientStreamingDataset`，避免将全量数据一次性读入内存。
   - **防泄漏**：针对单患者测试，实现了基于时间窗重排的**随机分层抽样 (Stratified Split)**，确保训练集和测试集无交集且类别平衡。

## 🚀 快速开始

### 环境依赖
建议使用 Python 3.8+，需安装以下主要包：
- `torch`
- `mne`
- `scipy`
- `numpy`
- `scikit-learn`
- `matplotlib`
- `seaborn`

### 运行流水线

项目提供了一个统一启动脚本 `run_pipeline.py`，支持两种运行模式：

#### 1. 快速验证模式 (单患者)
如果您想快速跑通流程或调试代码，可以只处理指定的单名患者（默认 `chb01`）：
```bash
python run_pipeline.py --mode single --patient chb01
```

#### 2. 全量数据模式
如果您想在整个 CHB-MIT 数据库上进行训练和测试：
```bash
python run_pipeline.py --mode all
```

*流水线将依次自动执行以下步骤：*
1. 解析标签并进行 `0.5-40Hz` 滤波、**全局** Z-score 归一化。
2. Max-Abs 频带降维与**固定绝对阈值 + 局部相对异常门控**脉冲编码，保存到 `processed_data/`。
3. 利用 `Memristor Reservoir` 提取特征并保存。
4. 按患者划分数据集并训练 **1D CNN 分类器**，打印原始和**平滑后**的 Precision, Recall, F1 分数。
5. 生成各种可视化图表到 `plots/`。

## 🧭 参数速查表

以下参数位于 [config.py](config.py)，是最常用的调参入口。

| 参数名 | 当前值 | 含义 | 调大后的典型影响 | 调小后的典型影响 |
|---|---:|---|---|---|
| `SAMPLING_RATE` | `256` | EEG 采样率（Hz） | 时间分辨率更高，计算开销增大 | 时间分辨率降低，动态可能丢失 |
| `WINDOW_SIZE_SEC` | `2` | 每个窗口时长（秒） | 单窗口上下文更完整，但响应更“慢” | 响应更快，但上下文不足 |
| `OVERLAP_SEC` | `1` | 相邻窗口重叠时长（秒） | 样本数量增加、时序更平滑 | 样本减少，边界信息可能粗糙 |
| `FIXED_THETA` | `0.8` | 固定绝对激发阈值 (全局标准差倍数) | 脉冲变得极度稀疏，仅抓取极强爆发 | 脉冲变密集，可能引入更多正常底噪 |
| `REFRACTORY_SAMPLES`| `3` | 不应期采样点数 (防止连续高频激发) | 脉冲更具有节奏感，防止RC层过快饱和 | 脉冲更密集，对瞬时高频更敏感 |
| `RELATIVE_STD_WINDOW` | `25` | 局部异常标准差窗口 | 对慢变化更稳健，但响应会更迟缓 | 对局部扰动更敏感，但更易受噪声影响 |
| `RELATIVE_THETA` | `1.2` | 局部相对异常门控阈值 | 只有更显著的局部异常才触发 spike，正常更干净 | pre-ictal 更容易变密，但也可能带入更多底噪 |
| `R_OFF` / `R_ON` | `391k`/`42k`| 忆阻器高低阻态 | 改变储层物理特性的上下限 | - |
| `DECAY_RATE` | `0.95` | RC 状态相对基线的衰减率 | 记忆保持时间更长（漏电慢） | 记忆消退极快（漏电快） |

## 📊 可视化与评估

运行结束后，您可以打开 `plots/` 文件夹查看以下结果：

1. **混淆矩阵 (`confusion_matrix.png`)**:
   展示应用滑动窗口平滑后，分类器在测试集上的最终表现，直观反映假阳性和真阳性的分布。

2. **t-SNE 聚类图 (`clustering_tsne_cnn.png` & `clustering_tsne.png`)**:
   强烈建议对比这两张图。`clustering_tsne.png` 展示了原始 RC 物理特征的可分性；而 `clustering_tsne_cnn.png` 展示了经过 1D CNN 深度解码后，隐层特征进一步被解开，形成了红（Pre-ictal）蓝（Normal）更清晰的聚类簇。

3. **脉冲图 (`spikes_seizure_top5.png` / `spikes_normal_top5.png`)**:
   这里虽然文件名沿用了历史命名，但在当前项目语境下，`spikes_seizure_top5.png` 应理解为 `Pre-ictal / Positive` 样本图。它展示了引入固定阈值、相对异常门控和不应期后，正类样本（更密、更连续）与正常样本（更稀疏、留白更多）在脉冲形态上的直观差异。

4. **RC状态演化散点图 (`rc_dynamics_scatter_*.png`)**:
   展示了忆阻器在接收到离散脉冲时，电流状态的连续上升与无脉冲时的漏积分回落过程（Leaky Integrator 效应）。在当前预警任务中，正类样本通常表现为更连续、更高亮的状态轨迹。

## 🧠 当前方法学总结

从 seizure detection 改到 pre-ictal warning 后，本项目最终采用的是一条**最小改动、最大保真**的路线：

- 不改 `4` 频带输入结构
- 不改 `RC` 主体和 `1024` 维特征结构
- 不改 `1D CNN` 主干
- 不引入人为规则化事件流
- 只在原始编码器上加入**局部相对异常门控**

换句话说，当前版本的核心不是“重新设计一套预测未来的复杂状态机”，而是：

**尽可能保留原版最有效的自然脉冲风格，让 pre-ictal / positive 样本在脉冲密度、RC 热图亮度和聚类结构上，都比 normal 更容易被忆阻器系统感知。**
